# AKP Architekten — Design Handoff Bundle

This repository is a **handoff bundle from Claude Design** containing high-fidelity prototypes for the AKP Architekten website. These files are intended as a design specification for implementation in a production-ready codebase.

## Project Overview

- **Purpose:** A high-fidelity "Architectural Atlas" dashboard prototype for a Berlin-based architecture firm.
- **Tech Stack (Prototype):** React (using global `window` objects), Vanilla CSS with CSS Variables, and structured JavaScript data.
- **Architecture:** The prototype is split into a data layer (`data.js`), a styling layer (`styles.css`), and functional UI components (`pages-a.jsx`, `pages-b.jsx`).

## Key Files & Directories

- `project/pages-a.jsx`: Primary design file. Contains React components for the Cover, Index, Büro, Leistungen, and Projekt-Archiv pages. **Read this file in full first.**
- `project/pages-b.jsx`: Supplemental design file. Contains React components for Wohnen, Gesundheit, Gewerbe, and Bestand pages.
- `project/data.js`: The "Single Source of Truth" (SSOT) for content, project metadata, and navigation structure.
- `project/styles.css`: Comprehensive CSS defining the "Magazine Stage" 3D effects, animations, typography, and responsive layouts.
- `project/assets/` & `project/uploads/`: Image assets and logos used in the designs.

## Implementation Guidelines

### 1. Pixel-Perfect Recreation
The primary mandate is to recreate these designs **pixel-perfectly** in the target production environment. Use the provided CSS for dimensions, colors, and layout rules.

### 2. Implementation Strategy
- **Framework Agnostic:** While the prototype uses React components, the goal is to match the visual output in whatever technology is chosen for the production site.
- **Data Integration:** Use the structures defined in `data.js` to drive the content of the implemented pages.
- **Styling:** Adhere strictly to the design tokens and layout patterns defined in `styles.css`. Pay special attention to the 3D "magazine" transitions and layered z-index animations.

### 3. Workflow
- **Research First:** Read `pages-a.jsx` and follow its imports/data dependencies before starting implementation.
- **Ambiguity:** If any design detail or behavior is unclear, clarify with the user before proceeding.
- **Rendering:** Do not attempt to run or render these files locally unless specifically requested. Use the source code as the primary specification.

## Building and Running
These files are prototypes and do not require a build step within this directory. They are specifications to be implemented elsewhere.
